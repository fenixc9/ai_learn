# ip rule vs ip route

#### You:
ip rule 和 ip route有什么区别

#### ChatGPT:
`ip rule` 和 `ip route` 都属于 Linux `iproute2` 工具集，但作用层级不同。

---

## 一句话区别

- **ip route**：定义“路怎么走”（路由表内容）
- **ip rule**：定义“用哪张路由表”（策略选择规则）

---

# 1️⃣ ip route —— 管理路由表

作用：\
向路由表中添加、删除或查看具体的路由条目。

本质：\
决定**目的地址匹配后，数据包下一跳怎么走**

### 示例

```bash
ip route add 192.168.1.0/24 via 10.0.0.1
```

含义：
去往 192.168.1.0/24 的流量，从 10.0.0.1 走。

查看路由：

```bash
ip route show
```

默认操作的是：

```
main 表（254）
```

---

# 2️⃣ ip rule —— 管理策略路由规则

作用：\
决定使用哪张路由表。

本质：\
在查找路由之前，先决定“查哪张表”。

Linux 支持多张路由表，例如：

| 表名      | 表号   |
| ------- | ---- |
| local   | 255  |
| main    | 254  |
| default | 253  |
| 自定义     | 自己定义 |

---

### 示例

```bash
ip rule add from 192.168.1.100 table 100
```

含义：

来源是 192.168.1.100 的流量\
不要查 main 表\
改去查 100 号路由表

---

然后你要在 table 100 里写路由：

```bash
ip route add default via 10.0.0.2 table 100
```

---

# 3️⃣ 数据包处理顺序

当数据包到来时：

```
① 先按 ip rule 从上到下匹配规则
② 找到对应 table
③ 再在该 table 里按 ip route 查路由
```

---

# 4️⃣ 对比总结

| 对比项    | ip rule              | ip route |
| ------ | -------------------- | -------- |
| 控制层级   | 更高层                  | 具体路由     |
| 决定什么   | 用哪张路由表               | 下一跳怎么走   |
| 是否支持策略 | 支持（源地址、fwmark、tos 等） | 不支持      |
| 是否支持多表 | 是                    | 单表内操作    |

---

# 5️⃣ 使用场景对比

## 仅单出口服务器

只需要：

```bash
ip route
```

不需要 `ip rule`

---

## 多出口服务器 / 双线接入

例如：

- 电信走 eth0
- 联通走 eth1
- 不同来源 IP 走不同出口

就必须：

```
ip rule + 多路由表
```

这叫：

**策略路由（Policy Routing）**

---

# 6️⃣ 类比理解

可以这样理解：

```
ip rule = 导航选择哪本地图册
ip route = 地图册里的具体路线
```

---

# 7️⃣ 查看当前规则

```bash
ip rule show
ip route show table all
```

---

# 结论

- `ip route` 是传统路由
- `ip rule` 是策略路由入口
- 多线路 / 多网卡 / 负载分流 必须用 ip rule
