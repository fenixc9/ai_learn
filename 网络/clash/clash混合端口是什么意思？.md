在 **Clash**（代理客户端）中，**“混合接口”**指的是：

> **HTTP 代理 和 SOCKS5 代理 共用同一个端口的监听方式。**

也就是一个端口同时支持两种代理协议。

---

## 工作原理

当你在配置文件中设置：

```yaml
mixed-port: 7890
```

Clash 会在 `7890` 端口上同时监听：

- HTTP 代理请求
    
- SOCKS5 代理请求
    

客户端连接后，Clash 会自动识别协议类型并处理。

---

## 和其他端口的区别

|配置项|作用|
|---|---|
|`port`|只开启 HTTP 代理|
|`socks-port`|只开启 SOCKS5 代理|
|`mixed-port`|同时开启 HTTP + SOCKS5（推荐）|

示例：

```yaml
port: 7890        # 仅 HTTP
socks-port: 7891  # 仅 SOCKS5
```

等价于：

```yaml
mixed-port: 7890  # 两种协议共用 7890
```

---

## 为什么推荐用混合接口？

- 只需要记一个端口
    
- 软件自动识别协议
    
- 配置更简单
    
- 避免端口冲突
    

大多数用户直接使用：

```yaml
mixed-port: 7890
```

即可。

---

## 什么时候不用混合接口？

- 某些旧设备或特殊环境只支持单协议
    
- 需要分别管理 HTTP 和 SOCKS5 端口
    
- 做精细化网络隔离
    

一般日常使用，**混合接口是默认推荐方式**。
