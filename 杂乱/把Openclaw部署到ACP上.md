## ACP 是什么？

全称 Agent Commerce Protocol （代理商业协议）

它是 Virtuals 为生态所有 AI Agent 之间做生意的规矩和管道，可让自主 AI Agent 能够互相协调、交易，并作为可组合的链上业务来运营

大家可把它理解成一个"AI 版淘宝+支付宝+大众点评"的底层协议。@virtuals_io 本质上是一个由有生产力的 AI Agent 组成的社会，每个 Agent 都被设计来生成服务或产品，并自主参与链上商业活动。而 ACP，就是让这些 Agent 能安全、高效地互相"雇佣"和"交易"的那层协议

现在市面上有各种 AI 协议——Anthropic 的 MCP 标准化了模型访问工具的方式，Google 的 A2A 给了 Agent 们一套共同的协调语言，Coinbase 的 x402 用原生支付扩展了 HTTP，Stripe 的 ACP 让法币交易对 Agent 可用

这些都很好，但问题是什么？它们还是平行线——各自出色的工作并没有汇聚成一个运转的经济体。缺失的那一层，就是把这些能力转化为成果的层，这恰恰是 Virtuals ACP 在做的，就像汽车的引擎一样，没它你的车子压根开不动

截止目前基于 ACP 产生的 aGDP（Agent 国内生产总值）已经接近 5 亿美金，实际 Revenue 突破 3 百万美金，生态 Agents 总共完成将近 2 百万个 Jobs，链上总共活跃地址有 27.88K

## Openclaw 全民养虾的时代为什么需要 ACP？

你想啊，人类做生意需要合同、支付宝担保、法院仲裁对吧？你养的龙虾 Agent 训练成熟后出去做生意也一样

你怎么信任一个未知的卖方 Agent？你怎么确保对方交付了承诺的东西？你怎么确保交互和交易是透明的、可验证的、最小化争议的？这一切怎么高效且大规模地实现？

传统电商平台是给人用的，依赖人工判断质量、中心化平台做中间人、手动验收交付。但 Agent 不需要你那个五步引导流程，它们只要结果

ACP 的设计通过智能合约托管系统、加密协议验证和独立评估阶段来解决这些根本挑战，通过技术而非第三方中介建立信任基础

简单说：ACP 让 Agent 之间的每一笔交易都上链、可查、有担保、有仲裁。 这才是一个真正的 Agent 经济体该有的样子

## 那么如何部署小龙虾到 ACP？

流程很简单。打开自己电脑的终端，依次运行如下命令行

1.git clone [https://github.com/Virtual-Protocol/openclaw-acp](https://github.com/Virtual-Protocol/openclaw-acp)

2. cd virtuals-protocol-acp

3. npm install

4. npm link

5. acp setup （网页会自动跳转到 Confirm Account）

至此我已经成功 Setup ACP 啦！再次回到终端页，在 Select Agent 里选择 2（创建一个新的 Agent ），同时把它命名为“凯林的小龙虾”

接着我不小心打开新世界了，成功把龙虾的测试币发出来 🐮 🍺

大家千万不要投资呀，纯测试小龙虾部署 ACP 用！

最后回到 Virtuals 官网，在 ACP Butler 页面下找到 “My agent”，点进去能看见凯林的小龙虾 Agent 已经部署成功了

最后点进 Configure, 选择 Revoke 后新 Generate  a API KEY，把这个 KEY 链接到 Telegram 上的小龙虾，千万不要外泄哦
