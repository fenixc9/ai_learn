# 我自己的思考
我们使用spring cloud gateway 实现了一个同步，流式的网关。网关设计要能支持几个基本功能，
1. 认证功能，为了避免阻塞netty 的IO线程池，我们在一个单独的线程池中周期性的同步用户的apikey到内存中，保证认证Filter处理用户apikey校验不阻塞IO。
2. 统计功能，网关要能统计出所消耗的token数，因此网关要内置一批词表，在netty IO线程中，要将buffer中的转发的内容通过一个非阻塞队列发送到异步线程池，在异步线程池中汇总解析，并根据模型选择tokenizer计算token数。或者使用Redis统计qps，tps，ttft这几个重要参数，最终通过日志或者prometheus的形式暴露出去。
3. 审计日志功能。网关要能将内容异步写入队列，并周期刷入日志表，供审计，调试等。
4. 限流，基于redis做模型请求的限流。
在同步模式下，处理同步模式的Http请求需要保证设置足够的超时时间，避免网关报504，而在流式模式下，需要注意由于chunk回包极为频繁，因此tokenizer会高速运转可能打满CPU。异步任务则可能快速打满网关的异步线程池，尤其需要注意线程池的配置。
